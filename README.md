# 調査用地図 v1

Googleスプレッドシートにまとめた調査記録を地図上に表示し、スマホで現在地と既存記録を見比べるための、閲覧専用Web地図です。調査ごとにスプレッドシートURLを登録しておくと、上部の調査名プルダウンから切り替えて表示できます。

## 想定ヘッダー

1行目に以下のヘッダーを書いてください。

```text
記録ID, 分類群, 記録の種類, 所蔵, 標本ID, 場所, 緯度, 経度, 年, 月, 日, 表示する日付, 備考
```

`記録の種類` には、以下のいずれかを入力します。

```text
タイプ産地
シノニマイズされた種のタイプ産地
文献記録
本調査
```

`表示する日付` には、ポップアップに表示する日付文字列を入力します。`年`、`月`、`日` は機械的な並べ替え・絞り込み用、`表示する日付` は人が読むための表記、という使い分けを想定しています。

```text
19 Jun. 1985
15–16 Apr. 1987
Jun. 1985
1985
```

## 主な機能

- 「使い方」から、このページの目的、必要なヘッダー、スプレッドシートの準備方法を確認できます。
- アプリ画面からGoogleスプレッドシートURLを登録し、端末内に保存できます。
- 登録した調査名をプルダウンで切り替えられます。
- Googleスプレッドシートの内容を読み込みます。
- 背景地図は「航空写真」と「淡色地図」を切り替えられます。
  - 航空写真：Esri World Imagery
  - 淡色地図：地理院淡色地図
- 調査データの追加・編集画面で、色分けに使用する項目、ポップアップタイトルに使用する項目、ポップアップ本文に表示する項目、フィルターに使用する項目を選択できます。色分け項目とポップアップタイトル項目は初期状態では未選択です。
- `記録の種類` ごとにマーカーの形を変えます。
  - タイプ産地：星
  - シノニマイズされた種のタイプ産地：四角
  - 本調査：中が塗りつぶされた丸
  - 文献記録：中が空の丸
- 「フィルター」から、スプレッドシートの各項目ごとに表示する記録を絞り込めます。
- フィルター適用後の表示件数をメニューに表示します。
- 画面上で近いマーカーを代表マーカー1個にまとめます。
- 近接グループに複数の色分け値が含まれる場合、代表マーカーをその値の色で radial（扇形）分割表示します。2値なら半円ずつ、3値なら1/3ずつ、4値以上も等分割します。各扇形は記録の種類に応じ、本調査などの標本記録は塗りつぶし、文献記録は中空表示になります。
- 代表マーカーを押すと、同じ統合マーカー内の記録に加え、画面上で近い別マーカーの記録もポップアップ内の「前へ」「次へ」で切り替えられます。切り替え時は、ポップアップの吹き出し先も切り替え先の表示マーカー中心へ移動します。
- 地図の拡大・縮小中はマーカーを一時的に非表示にし、ズーム終了後に近接判定を再計算して再表示します。
- 現在地ボタンを表示します。
- 「地図表示」欄の「更新」ボタンで更新します。

## ポップアップ表示

ポップアップは、地図上のマーカーをクリックしたときに、マーカーより前面に表示される情報欄です。透明な余白部分は下のマーカーのクリックを妨げないようにしています。ポップアップタイトルに使用する項目、本文の表示項目、本文の表示順は、調査データの追加・編集画面で変更できます。本文の初期状態は次の順番です。

```text
記録の種類
場所
緯度, 経度 [Google Map]
表示する日付
所蔵
標本ID
記録ID
備考
```

ポップアップタイトル項目を選択している場合は、その項目の値をポップアップのタイトルとして表示します。ポップアップタイトル項目を選択していない場合は、タイトルを表示しません。色分け項目とポップアップタイトル項目は別々に選択できます。タイトル項目でも、本文でチェックされていれば本文にも表示します。タイトル横の小さなマーカーには、地図上のマーカーと同じく、記録の種類に応じた形と塗りつぶしを反映します。

`Google Map` は、その座標を Google Maps で開くリンクです。

## フィルター

上部メニューの「フィルター」から、スプレッドシートの各項目ごとに表示する記録を絞り込めます。

対象は、調査データの追加・編集画面で選択した項目です。列が増減した場合は、「列を読み込む」から1行目のヘッダーを再読み込みして設定できます。

同じ項目内では OR 条件、異なる項目間では AND 条件になります。たとえば、分類群で A と B、記録の種類で「本調査」を選んだ場合は、「分類群が A または B」かつ「記録の種類が本調査」の記録だけを表示します。

フィルターは調査ごとに、この端末の `localStorage` に保存されます。別の調査へ切り替えた場合は、その調査用のフィルター設定が使われます。

フィルター適用後は、表示対象の記録だけで選択した色分け項目の色、近接グループ、複数値の radial（扇形）分割、凡例を再計算します。凡例には現在表示中の色分け値と記録の種類だけを表示します。

## 色分け項目と色の割り当て

色分け項目を選択すると、その項目の値に基づいて現在表示中の記録へ色を自動で割り当てます。色分け項目を選択しない場合、マーカーは灰色で表示されます。

固定距離や現在ズームは使わず、表示中の各記録について「最も近い別の値」を近い順に複数探します。同じ近傍に出やすい値どうしでは、同じ色グループをできるだけ避け、色差の大きい組み合わせを優先します。

値の数が色候補数以内の場合は、表示中の値どうしで同じ色を再利用しません。表示分類群が少ない場合は、その範囲で互いに十分離れた高コントラスト色を優先して割り当てます。

`config.js` の `COLOR_NEAREST_DIFFERENT_TAXA` で、各記録から何種類の近い別分類群を見るかを指定できます。初期値は `5` です。

## 近接マーカーの扱い

`config.js` の `PROXIMITY_PIXELS` で、近接記録とみなす画面上の距離を指定します。初期値は `10` です。

代表マーカーの外形と前面表示は、以下の優先度で決めます。複数の色分け値が含まれる場合は、代表マーカー内をその値の色で radial（扇形）分割表示し、文献記録の扇形は中空、本調査などの標本記録の扇形は塗りつぶしで表示します。

```text
塗りつぶし星 > 中空星 > 塗りつぶし四角 > 中空四角 > 塗りつぶし丸 > 中空丸
```

重なったマーカーでは、優先度の高いマーカーを前面に出します。ポップアップの近接マーカー切り替え候補も、クリックしたマーカーを最初にしたうえで、近い別マーカーをこの優先順位に基づいて並べます。

`config.js` の `POPUP_NEARBY_MARKER_PIXELS` で、ポップアップ切り替え候補に含める近接マーカーの画面上の距離を指定します。初期値は `10` です。`detail.zip` と同様に、近接記録の統合距離と同程度にしています。

## 使い方

上部メニューの「ヘルプ」欄にある「使い方」から、以下を確認できます。

- このページの目的
- 基本的な使い方
- 必要な列
- フィルターの使い方
- `記録の種類` の記入例
- `表示する日付` の記入例

「追加」や「編集」の入力フォームには、調査名・GoogleスプレッドシートURL・シート名だけを記入します。

## 調査名の登録

この版では、アプリ本体に特定のスプレッドシートを固定登録していません。初回起動時は「調査名未登録」と表示されます。

上部メニューの「調査データ」欄にある「追加」から入力フォームを開き、以下をまとめて登録してください。「編集」でも同じフォームで内容を修正できます。スプレッドシートの準備や必要な列については、メニューの「使い方」を確認してください。

```text
調査名：四国のXystodesmus、関東のParadoxosomatidae など
GoogleスプレッドシートURL：通常の共有URL
シート名：通常は シート1。英語環境で作成した場合は Sheet1 のことがあります。
```

登録情報は、その端末・そのブラウザの `localStorage` に保存されます。別の端末では再登録が必要です。ブラウザのサイトデータを削除すると、登録一覧も削除されます。

スプレッドシートは、このアプリのブラウザから読み込める状態にしておく必要があります。通常のGoogleスプレッドシートURLを登録する場合は、リンクを知っている人が閲覧できる共有設定にしてください。公開CSV URLを登録する場合は、CSVとしてウェブに公開されたURLを使用してください。

登録後は、上部メニューの「調査データ」欄にある「調査名」プルダウンで切り替えられます。「編集」は入力フォームで端末内の登録情報を修正します。「削除」は端末内の登録情報だけを削除します。スプレッドシート本体は変更・削除しません。

## 設定

通常は `config.js` の次の項目だけ確認してください。

```javascript
DEFAULT_SHEET_NAME: "シート1",
INITIAL_BASEMAP: "detail",
PROXIMITY_PIXELS: 10,
COLOR_NEAREST_DIFFERENT_TAXA: 5,
```

GoogleスプレッドシートURLは、アプリ画面の「追加」から登録します。公開CSV URLを登録することもできます。

## 公開方法

現在地表示は、スマホブラウザではHTTPS上での利用が基本です。
GitHub Pages、Cloudflare Pages、Netlifyなどに配置して使うことを想定しています。

## detail.zip から反映した点

- 学名の自動斜体化は `detail.zip` の `app-labels.js` の考え方に合わせています。
  - `ord.`、`fam.`、`gen.` を含む場合は通常体
  - `sp.` は通常体
  - `cf.`、`aff.` は通常体
  - それ以外の属名・種小名などを斜体
  - `和名 / 学名` 形式では学名側だけを整形
- 背景地図は、`detail.zip` の航空写真に合わせた Esri World Imagery と、地理院淡色地図を切り替えられるようにしています。

## 調査の共有

上部メニューの「調査データ」欄にある「共有」から、現在選択中の調査結果の共有リンクを作成できます。

共有リンクには、以下の情報だけを含めます。

```text
調査名
スプレッドシートIDまたは公開CSV URL
シート名
背景地図
```

共有画面には、共有リンクとは別に、編集元のスプレッドシートまたは公開CSVのリンクも表示されます。

共有には、現在見ている中心座標や現在のズームは含めません。リンクを開いた側では、スプレッドシートを読み込んだあと、全プロットが地図内に収まるように自動表示します。

共有リンクから開いた調査は、最初は閲覧のみです。端末内の登録一覧には自動では追加されません。

この端末の調査一覧に残したい場合は、上部メニューの「調査データ」欄にある「登録」を押してください。登録すると、次回から調査名プルダウンで選択できます。スプレッドシートの記録データそのものは保存されません。

すでに同じスプレッドシートとシート名の調査が登録されている場合は、既存の登録を選択します。

注意：
スプレッドシートがリンクを知っている人に閲覧可能、またはCSVとしてウェブに公開されている必要があります。
リンクを受け取った人は、記録の位置情報と各項目を閲覧できます。
希少種、未公開データを含む場合は、共有範囲に注意してください。
