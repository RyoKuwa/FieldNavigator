# 調査結果地図 v1

Googleスプレッドシートにまとめた記録データを地図上に表示し、スマホで現在地と既存記録を見比べるための、閲覧専用Web地図です。調査結果ごとにスプレッドシートURLを登録しておくと、上部のプルダウンから切り替えて表示できます。

## このアプリでできること

- Googleスプレッドシートまたは公開CSVの記録を地図上に表示します。
- データを端末内に登録し、プルダウンから切り替えられます。
- 位置情報に使用する列、色分けに使用する項目、ポップアップタイトルに使用する項目、ポップアップ本文に表示する項目、フィルターに使用する項目を、データごとに選択できます。
- 背景地図は「航空写真」と「淡色地図」を切り替えられます。
- 現在地ボタンを表示します。
- 現在地ボタンの下に、表示中の記録範囲へ移動するフォーカスボタンを表示します。
- 「地図表示」欄の「更新」ボタンでスプレッドシートの内容を再読み込みします。
- 近い位置の記録は代表マーカーとしてまとめ、ポップアップ内の「前へ」「次へ」で記録を切り替えられます。
- 共有リンクを作成し、同じスプレッドシートと表示設定で地図を開けるようにできます。

## 1. スプレッドシートを準備する

まず、記録データを入力したGoogleスプレッドシートを用意します。1行目には列名を入力してください。

### 位置情報に使用する列

地図に記録を表示するには、緯度と経度の情報が必要です。スプレッドシートを追加または編集するときに、どの列を緯度として使うか、どの列を経度として使うかを選択できます。

| 必要な情報 | 説明 |
|---|---|
| 緯度 | 記録地点の緯度です。10進数で入力します。 |
| 経度 | 記録地点の経度です。10進数で入力します。 |

`緯度` / `経度`、`Latitude` / `Longitude`、`lat` / `lon`、`decimalLatitude` / `decimalLongitude` などは、大文字・小文字の違いを無視して自動推定します。`採集緯度` や `WGS84_Longitude` のように、列名にこれらの語が含まれる場合も候補にします。推定結果は後から変更できます。

### 推奨ヘッダー

推奨ヘッダーは、このアプリの標準機能で扱いやすいように想定している列名の例です。必須ではありませんが、列名をそろえておくと、ポップアップ、フィルター、色分け、マーカー形状の設定が分かりやすくなります。

| 列名 | 用途 |
|---|---|
| 記録ID | 各記録を識別するためのIDです。 |
| 分類群 | 種名・属名など、記録対象の分類群です。 |
| 記録の種類 | 「タイプ産地」、「文献記録」、「作成者の記録」など、記録の種類です。マーカー形状に使われます。 |
| 所蔵 | 標本の所蔵先です。 |
| 標本ID | 標本番号・登録番号などです。 |
| 場所 | 採集地・記録地名です。 |
| 緯度 | 地図表示に使用する緯度です。 |
| 経度 | 地図表示に使用する経度です。 |
| 年 | 記録年です。フィルターや整理に使えます。 |
| 月 | 記録月です。フィルターや整理に使えます。 |
| 日 | 記録日です。フィルターや整理に使えます。 |
| 日付 | ポップアップなどに表示する日付表記です。 |
| 備考 | 補足情報です。 |

Googleスプレッドシートの1行目にそのまま貼り付ける場合は、次の行をコピーしてA1セルに貼り付けてください。タブ区切りなので、各項目が別々の列に入ります。

```text
記録ID	分類群	記録の種類	所蔵	標本ID	場所	緯度	経度	年	月	日	日付	備考
```

### `記録の種類` の入力例

`記録の種類` には、マーカー形状を切り替えるため、次のような値を入力します。

```text
タイプ産地
シノニマイズされた種のタイプ産地
文献記録
作成者の記録（または、本調査）
不確かな記録
その他の値
```

対応するマーカー形状は次の通りです。

| 記録の種類 | マーカー形状 |
|---|---|
| タイプ産地 | 星 |
| シノニマイズされた種のタイプ産地 | 四角 |
| 作成者の記録（または、本調査） | 塗りつぶし丸 |
| 文献記録 | 中空丸 |
| 不確かな記録 | ✕印 |
| その他の値 | △ |


### `日付` の入力例

`日付` には、ポップアップに表示する日付文字列を入力します。`年`、`月`、`日` は機械的な並べ替え・絞り込み用、`日付` は人が読むための表記、という使い分けを想定しています。

```text
19 Jun. 1985
15–16 Apr. 1987
Jun. 1985
1985
```

## 2. スプレッドシートを共有可能にする

スプレッドシートは、このアプリのブラウザから読み込める状態にしておく必要があります。通常のGoogleスプレッドシートURLを登録する場合は、リンクを知っている人が閲覧できる共有設定にしてください。公開CSV URLを登録する場合は、CSVとしてウェブに公開されたURLを使用してください。

希少種や未発表データなど、公開範囲に注意が必要な位置情報を含む場合は、共有設定を慎重に確認してください。

## 3. アプリにデータを追加する

この版では、アプリ本体に特定のスプレッドシートを固定登録していません。初回起動時は「調査名未登録」と表示されます。

上部メニューの「データ」欄にある「追加」から入力フォームを開き、以下を登録してください。「編集」でも同じフォームで内容を修正できます。

```text
調査名：四国のXystodesmus、関東のParadoxosomatidae など
GoogleスプレッドシートURL：通常の共有URL
シート名：通常は シート1。英語環境で作成した場合は Sheet1 のことがあります。
```

入力後、「列を読み込む」を押して1行目の列名を読み込みます。緯度として使う列と経度として使う列を確認し、必要に応じて変更してください。そのうえで、色分け、ポップアップ、フィルターに使う項目を選択します。

登録情報は、その端末・そのブラウザの `localStorage` に保存されます。別の端末では再登録が必要です。ブラウザのサイトデータを削除すると、登録一覧も削除されます。

登録後は、上部メニューの「データ」欄にある「調査名」プルダウンで切り替えられます。「削除」は端末内の登録情報だけを削除します。スプレッドシート本体は変更・削除しません。

## 4. 表示項目を設定する

スプレッドシートを追加または編集するときに、地図上での表示方法を設定できます。

| 設定項目 | 説明 |
|---|---|
| 緯度として使う列・経度として使う列 | 地図上に記録を表示するために使う位置情報の列です。列名は自由に選べます。 |
| 色分けに使用する項目 | マーカーの色を分けるために使う列です。未選択の場合、マーカーは灰色で表示されます。 |
| ポップアップタイトルに使用する項目 | マーカーをクリックしたときに表示されるポップアップの見出しに使う列です。未選択の場合、タイトルは表示されません。 |
| ポップアップ本文に表示する項目 | ポップアップ本文に表示する列です。表示順も変更できます。 |
| フィルターに使用する項目 | 地図上の記録を絞り込むために使う列です。 |

ポップアップは、地図上のマーカーをクリックしたときに表示される情報欄です。タイトル項目でも、本文でチェックされていれば本文にも表示されます。

地図表示時には、位置情報、色分け、ポップアップ、フィルター、マーカー形状に必要な列だけを読み込みます。大きいスプレッドシートでは、使わないポップアップ項目やフィルター項目を外すと読み込みが速くなる場合があります。

読み込み中は、列情報の取得、データ読み込み、地図用データ作成、マーカー描画の段階を表示します。時間がかかる場合は「中止」ボタンで読み込みを止められます。

## 5. 地図を操作する

### 現在地ボタンとフォーカスボタン

右上の現在地ボタンで、端末の現在地を表示できます。現在地ボタンの下にあるフォーカスボタンを押すと、現在のフィルター条件で表示中の記録全体が入る範囲へ地図が移動します。

### マーカー

マーカーの形は、`記録の種類` に応じて変わります。タイプ産地など重要な記録が見つけやすいように、星、四角、丸、✕印、△の順で優先して表示されます。同じ形の場合は、塗りつぶしのマーカーが中空のマーカーより前面に表示されます。

クリック判定も、外側の正方形ではなく、丸・四角・星の見た目に近い範囲に合わせています。近くにあるマーカーでは、見えていない四角い余白をクリック対象にしないことで、別のマーカーが開きにくくなります。

近い位置の記録は、画面上の距離に基づいて代表マーカー1個にまとめます。複数の色分け値が含まれる場合、代表マーカーをその値の色で radial（扇形）分割表示します。2値なら半円ずつ、3値なら1/3ずつ、4値以上も等分割します。

代表マーカーの外形と前面表示は、以下の優先度で決めます。

```text
塗りつぶし星 > 中空星 > 塗りつぶし四角 > 中空四角 > 塗りつぶし丸 > 中空丸
```

### ポップアップ

ポップアップは、地図上のマーカーをクリックしたときに、マーカーより前面に表示される情報欄です。別のマーカーを開く場合は、そのマーカー本体を直接クリックします。ポップアップに遮られてマーカー本体を直接クリックできなかった場合は、誤って近くの別マーカーを開かないよう、クリック位置からマーカーを推定して開く処理は行いません。

代表マーカーを押すと、同じ統合マーカー内の記録に加え、画面上で近い別マーカーの記録もポップアップ内の「前へ」「次へ」で切り替えられます。切り替え時は、ポップアップの吹き出し先も切り替え先の表示マーカー中心へ移動します。

タイトル横の小さなマーカーには、地図上のマーカーと同じく、記録の種類に応じた形と塗りつぶしを反映します。

`緯度, 経度 [Google Map]` は、その座標をGoogle Mapsで開くリンクです。

### フィルター

上部メニューの「フィルター」から、スプレッドシートの各項目ごとに表示する記録を絞り込めます。

対象は、データの追加・編集画面で選択した項目です。列が増減した場合は、「列を読み込む」から1行目のヘッダーを再読み込みして設定できます。

同じ項目内ではOR条件、異なる項目間ではAND条件になります。たとえば、分類群でAとB、記録の種類で「作成者の記録」を選んだ場合は、「分類群がAまたはB」かつ「記録の種類が作成者の記録」の記録だけを表示します。検索欄で候補を絞り込んだ場合、「表示項目をすべて選択」「表示項目を解除」は、現在表示されている候補だけに適用されます。

フィルターは調査結果ごとに、この端末の `localStorage` に保存されます。別の調査結果へ切り替えた場合は、その調査結果用のフィルター設定が使われます。

フィルター適用後は、表示対象の記録だけで選択した色分け項目の色、近接グループ、複数値の radial（扇形）分割、凡例を再計算します。凡例には現在表示中の色分け値と記録の種類だけを表示します。

## 6. 色分け項目と色の割り当て

色分け項目を選択すると、その項目の値に基づいて現在表示中の記録へ色を自動で割り当てます。色分け項目を選択しない場合、マーカーは灰色で表示されます。

固定距離や現在ズームは使わず、表示中の各記録について「最も近い別の値」を近い順に複数探します。同じ近傍に出やすい値どうしでは、同じ色グループをできるだけ避け、色差の大きい組み合わせを優先します。

値の数が色候補数以内の場合は、表示中の値どうしで同じ色を再利用しません。表示分類群が少ない場合は、その範囲で互いに十分離れた高コントラスト色を優先して割り当てます。

`config.js` の `COLOR_NEAREST_DIFFERENT_TAXA` で、各記録から何種類の近い別分類群を見るかを指定できます。初期値は `5` です。

## 7. 近接マーカーの設定

`config.js` の `PROXIMITY_PIXELS` で、近接記録とみなす画面上の距離を指定します。初期値は `10` です。

`config.js` の `POPUP_NEARBY_MARKER_PIXELS` で、ポップアップ切り替え候補に含める近接マーカーの画面上の距離を指定します。初期値は `10` です。近接記録の統合距離と同程度にしています。

## 8. 共有リンク

上部メニューの「データ」欄にある「共有」から、現在選択中の調査結果の共有リンクを作成できます。

共有リンクには、以下の情報を含めます。

```text
調査名
スプレッドシートIDまたは公開CSV URL
シート名
背景地図
色分けに使用する項目
ポップアップタイトルに使用する項目
ポップアップ本文に表示する項目
フィルターに使用する項目
```

共有画面には、共有リンクとは別に、編集元のスプレッドシートまたは公開CSVのリンクも表示されます。

共有には、現在見ている中心座標や現在のズームは含めません。リンクを開いた側では、スプレッドシートを読み込んだあと、全プロットが地図内に収まるように自動表示します。

共有リンクで開いた調査結果を一覧として保存したい場合は、上部メニューの「データ」欄にある「登録」を押してください。登録すると、次回からプルダウンで選択できます。スプレッドシートの記録データそのものは保存されません。

注意：リンクを受け取った人は、記録の位置情報と各項目を閲覧できます。希少種、未公開データを含む場合は、共有範囲に注意してください。

## 9. 設定

通常は `config.js` の次の項目だけ確認してください。

```javascript
DEFAULT_SHEET_NAME: "シート1",
INITIAL_BASEMAP: "detail",
PROXIMITY_PIXELS: 10,
POPUP_NEARBY_MARKER_PIXELS: 10,
COLOR_NEAREST_DIFFERENT_TAXA: 5,
```

GoogleスプレッドシートURLは、アプリ画面の「追加」から登録します。公開CSV URLを登録することもできます。

## 10. 公開方法

現在地表示は、スマホブラウザではHTTPS上での利用が基本です。GitHub Pages、Cloudflare Pages、Netlifyなどに配置して使うことを想定しています。

## 11. detail.zip から反映した点

- 学名の自動斜体化は `detail.zip` の `app-labels.js` の考え方に合わせています。
  - `ord.`、`fam.`、`gen.` を含む場合は通常体
  - `sp.` は通常体
  - `cf.`、`aff.` は通常体
  - それ以外の属名・種小名などを斜体
  - `和名 / 学名` 形式では学名側だけを整形
- 背景地図は、`detail.zip` の航空写真に合わせたEsri World Imageryと、地理院淡色地図を切り替えられるようにしています。

## 12. 画面表示

上部の操作メニューは右上の丸い「×」アイコンで折りたためます。閉じた後は左上の丸い「☰」アイコンで再表示できます。


凡例はマーカーより前面に表示されます。メニューから開く入力画面や選択画面は、凡例やポップアップよりさらに前面に表示されます。
